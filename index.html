Selenium Automation Testing

Open Repository

In this project, I developed a robust automation testing framework utilizing Cucumber and Selenium. The framework is designed for end-to-end testing of web applications, ensuring that functionality is maintained across various scenarios. To maintain clean, modular, and scalable code, I implemented a Page Object Model (POM) structure, which organizes the codebase and separates test logic from page interactions.

Key Features
  • Behavior-Driven Development (BDD): By using Cucumber, I wrote test scenarios in a natural language format, making them accessible and understandable for non-technical stakeholders.

  • Selenium WebDriver Integration: Automated browser interactions for various test cases, from user login to complex form submissions, ensuring the application’s UI remains functional under different conditions.

  • Page Object Model (POM): Implemented POM to enhance code maintainability and scalability, allowing for easy updates to page elements and ensuring reusable code

  • Cross-Browser Testing: Configured the framework to test across multiple browsers, ensuring consistent behavior and performance.
